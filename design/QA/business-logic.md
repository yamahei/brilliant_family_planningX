品質管理：ビジネスロジック
==========================

本ドキュメントでは、ビジネスロジック（`Simulator`等）の品質を保証するためのテスト観点を定義する。
ビジネスロジックは、コアロジック（単体ルール）を束ねたイベントの評価と、銀行の残高シミュレーションを責務とする。

1. 複数ルールの組み合わせ（AND条件）
--------------------------------

1つの `Event` に設定された複数（テストでは2つ）の `Rule` が、すべて満たされた（AND）場合のみイベントが発生すること。

| パターン | ルールAの判定結果 (Not適用前) | ルールAのNotフラグ | ルールBの判定結果 (Not適用前) | ルールBのNotフラグ | イベント発生結果 |
|---|---|---|---|---|---|
| 1 | True | False | True | False | 発生する |
| 2 | True | False | False | False | 発生しない |
| 3 | False | False | True | False | 発生しない |
| 4 | False | False | False | False | 発生しない |
| 5 | True | True (反転) | True | False | 発生しない |
| 6 | True | True (反転) | False | False | 発生しない |
| 7 | False | True (反転) | True | False | 発生する |
| 8 | False | True (反転) | False | False | 発生しない |
| 9 | True | False | True | True (反転) | 発生しない |
| 10| True | False | False | True (反転) | 発生する |
| 11| False | False | True | True (反転) | 発生しない |
| 12| False | False | False | True (反転) | 発生しない |
| 13| True | True (反転) | True | True (反転) | 発生しない |
| 14| True | True (反転) | False | True (反転) | 発生しない |
| 15| False | True (反転) | True | True (反転) | 発生しない |
| 16| False | True (反転) | False | True (反転) | 発生する |


2. 有効期間（FromTo）の重なり判定
-------------------------------

`Entity` と `Event` にはそれぞれ個別の有効期間（FromTo）が存在する。
イベントが発生するのは、**「Entityの期間内」かつ「Eventの期間内」が重なっている共通期間のみ**である。

| パターン | 説明 | 期待されるイベント発生期間（AND条件の重なり） |
|---|---|---|
| 1 | Entity期間とEvent期間が完全に同一 | Entity期間(==Event期間) |
| 2 | Entity期間の中にEvent期間が完全に含まれる | Event期間 |
| 3 | Event期間の中にEntity期間が完全に含まれる | Entity期間 |
| 4 | Entityの終了より前に、Eventが開始して重なる | Event開始 〜 Entity終了 |
| 5 | Eventの終了より前に、Entityが開始して重なる | Entity開始 〜 Event終了 |
| 6 | Entity期間の終了後にEvent期間が開始する | 発生しない（重なりなし） |
| 7 | Event期間の終了後にEntity期間が開始する | 発生しない（重なりなし） |


3. 銀行別の残高計算（シミュレーション）
------------------------------------

実際の銀行口座残高の計算におけるエッジケースやルールを検証する。

### 3-1. 実績ログの起点と過去の空欄化
- **計算の順方向性**: 指定した月の残高は、その銀行の「最も新しい過去の `ActualLog`（実績ログ）」を起点として計算されること。
- **最古ログ以前の計算除外**: 銀行の「最も古い `ActualLog`」より過去の年月については、累積計算を行わず `BankBalanceLog.balance` が `null` として出力されること。（発生したイベント自体は `EventLog` に記録される）

### 3-2. 銀行への振り分けとフォールバック
- **独立性**: Eventで指定された銀行（`BankId`）の残高のみが変動し、他の銀行の残高には影響を与えないこと。
- **デフォルト銀行への適用**: Eventの `BankId` が `null` の場合、全額がデフォルト銀行（`DefaultBankId`）で計算されること。
- **削除済み銀行のフォールバック**: Eventに紐付いていた銀行が既にマスター（`Banks`）から削除されている場合、論理的に `null` と見なし、デフォルト銀行で計算されること。

### 3-3. 総計の算出
- **総残高の集計**: 複数銀行の合算（`totalBalance` 等を上位で集計する場合）において、`balance` が `null` の銀行は 0円（または計算から除外）として正しく扱われること。
- **マイナス残高許容**: イベントの支出により銀行残高がマイナス（0未満）になっても、そのままマイナスの値として正常に計算・出力されること。
